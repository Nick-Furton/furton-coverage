# Furton Coverage

Automated earnings-coverage desk on ~10 names, built with Claude Code + the Daloopa plugin.
Read PLAN.md before doing anything — it defines the phases, session pairings, and path ownership.

## Rules for every session

- **You are one of a pair.** Another Claude Code session may be working in this repo right now.
  Write ONLY inside the paths your kickoff prompt says you own. Shared proposals go in
  `notes/_inbox/`, never directly into another session's files.
- **Commits:** prefix with your session tag (`[S3] ...`), stage only your owned files (never
  `git add -A`). On push/commit races: `git pull --rebase`, retry.
- **End-of-session:** commit, push, append a 3-line handoff to `notes/_inbox/handoffs.md`
  (what you did / what's unfinished / what the next session must know).
- **Fail loud.** Missing data, a Daloopa error, or a quota limit is a stop-and-report, never a
  silently degraded output. (Same philosophy as Furton Research's vote-integrity work.)

## Conventions

- Notes: `notes/<TICKER>/<YYYY-MM-DD>_<type>.md`, type ∈ preview | flash | review | initiation.
  Required frontmatter: `ticker, type, event_date, published_at`.
- Models: `models/<TICKER>/<TICKER>_model_<YYYY-MM-DD>.xlsx`.
- Scorecard entries always carry `source_note` pointing at the note that made the call.
- `docs/` is generated by `scripts/build_site.py` — never hand-edit it.
- Site preview: port **8802** (8801, 8899, 8765 are taken by other Furton projects).
- Every published page carries the educational / not-investment-advice disclaimer.

## Environment notes (Windows)

- Python: run via `py`; set `PYTHONUTF8=1` (console is cp1252 — ✓/→ symbols crash otherwise).
- Word/PDF: no LibreOffice; convert docx→PDF via Word COM in a PowerShell job.
- Daloopa is an interactively-authenticated MCP connector — assume it is ABSENT in
  headless/scheduled runs unless RUNBOOK.md says otherwise.
- `gh` is installed and authed as Nick-Furton.
